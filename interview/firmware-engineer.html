Sample Interview Questions

Q1
How do you debug firmware running on embedded hardware?
Tip: Discuss JTAG debugging, logic analyzers, oscilloscopes, printf over UART, and stepping through code on target hardware.
Q2
Describe your experience with real-time operating systems.
Tip: Mention FreeRTOS, Zephyr, or bare-metal scheduling. Discuss task prioritization, interrupt handling, and timing constraints.
Q3
How do you optimize firmware for limited memory and processing power?
Tip: Show experience with code size optimization, memory-mapped I/O, DMA, and efficient data structures for constrained environments.
Q4
What is your experience with communication protocols for embedded systems?
Tip: Discuss SPI, I2C, UART, CAN bus. Show understanding of timing, addressing, and when to use each protocol.
